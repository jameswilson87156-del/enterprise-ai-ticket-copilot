# 演示脚本

## 演示目标

用 3 到 5 分钟说明这个项目不是普通后台，也不是聊天页，而是一个企业工单辅助处理 Copilot 工作台：员工提交问题后，系统用本地规则完成分类、知识匹配和模板化建议草稿生成，支持人员再人工确认处理与沉淀知识。这里的 Copilot 指辅助处理工作台，不代表接入真实大模型。

## 准备步骤

只演示前端交互时，可以直接运行：

```bash
cd frontend
npm run dev:demo
```

Demo 模式使用浏览器内存数据，刷新后会重置，不会写入真实企业系统。

需要演示 MySQL 闭环时：

1. 初始化数据库：

```bash
mysql -uroot -p < backend/src/main/resources/schema.sql
mysql -uroot -p enterprise_ai_ticket_copilot < backend/src/main/resources/demo-data.sql
```

2. 启动后端：

```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

3. 启动前端：

```bash
cd frontend
npm run dev
```

## 演示路径

1. 打开 `http://localhost:5173`，展示浅色标题区、本地 Demo 模式、搜索筛选和五个轻量指标。
2. 在工单队列中选择 `DEMO-0001`，说明队列里能直接看到优先级、状态、分类和规则置信度。
3. 展示当前工单详情：问题摘要、系统上下文、原始日志、处理记录和状态时间线。
4. 展示规则分析与建议草稿卡片：分类原因、置信度、知识库命中、排查步骤、回复建议和风险提示。
5. 强调所有建议都显示“待人工确认”，系统不做自动生产操作。
6. 选择 `DEMO-0007`，点击“生成知识草稿”，展示知识草稿状态为 `DRAFT`。
7. 点击“人工确认入库”，展示工单进入“已沉淀”，知识条目变为 `PUBLISHED`。
8. 提交一个新工单，例如 Redis 连接失败，展示数据库闭环：新工单进入队列，系统生成规则分析和建议草稿，人工流转状态。
9. 搜索 `KB-REDIS-CONN` 或切换“已沉淀”筛选，说明搜索和筛选不是静态装饰。

## 推荐讲解词

这个项目的重点不是把 ChatGPT 接进工单系统，而是把“辅助判断”约束在企业内部可审计的流程里。规则引擎负责分类，知识库评分负责复用历史经验，模板负责生成处理建议草稿，人工确认负责控制风险。第 2 阶段已经跑通 MySQL 数据闭环；如果后续评估企业私有模型或检索增强，也必须保留人工确认边界。
