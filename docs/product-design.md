# 产品设计

## 产品定位

Enterprise AI Ticket Copilot 是企业内部工单辅助处理与知识库工作台，服务对象是员工、IT 支持、运维、业务支持人员。Copilot 在本项目中表示辅助处理工作台，不代表接入真实大模型。它的核心价值不是“替人自动处理一切”，而是把问题上下文、知识库经验和模板化建议草稿组织到同一个工作台中，帮助人工支持人员更快、更稳地判断。

## 核心流程

1. 员工提交问题，包含描述、影响范围、系统信息、日志和业务上下文。
2. 规则引擎进行辅助分类，给出置信度和可能的问题域。
3. 系统匹配知识库，展示命中文章、相关度、负责人和最近校验时间。
4. 系统根据模板生成排查步骤、回复建议和风险提示草稿。
5. 人工确认建议，决定是否采纳、修改、退回补充信息或升级处理。
6. 工单进入状态流转，保留每次人工和规则/模板辅助动作的审计记录。
7. 解决后的工单沉淀为知识候选，由人工审核后进入知识库。

## 第 0 阶段页面结构

工作台第一屏即进入可用场景：

- 顶部指标：待处理工单、规则命中率、知识关联率、今日沉淀知识数。
- 左侧队列：展示状态、优先级、分类、更新时间、规则置信度。
- 中间详情：展示问题描述、系统上下文、错误日志、业务上下文。
- 右侧建议草稿：展示分类、置信度、知识库命中、排查步骤、回复建议、风险提示和人工确认按钮。
- 底部时间线：展示提交、规则引擎辅助分类、知识库匹配、人工接手等状态流转。

## 人工确认原则

所有规则/模板输出均是建议草稿：

- 系统不自动回滚、不自动授权、不自动变更生产配置。
- 系统不直接改变工单状态。
- 涉及权限、安全、生产故障、客户影响的动作必须由人工确认。
- 建议草稿需要保留置信度、知识来源和风险提示，便于人工判断。

## 后续扩展

- 工单提交与附件管理。
- 知识库检索口径优化，例如更细的关键词权重、分类权重和人工反馈统计。
- 角色权限：员工、支持专员、SRE、知识库审核员、管理员。
- 状态机与 SLA 计时。
- 规则分析结果版本化与人工采纳率统计。
- 已解决工单转知识候选。
