# 第 2 阶段验收清单

## 数据库

- [x] `support_ticket` 映射工单主数据。
- [x] `knowledge_article` 支持发布知识和草稿知识。
- [x] `ticket_ai_analysis` 保存分类、原因、置信度、知识命中、建议和风险提示。
- [x] `generation_record` 保存规则/模板生成记录。
- [x] `ticket_status_history` 保存状态流转历史。
- [x] `demo-data.sql` 提供 8 条企业内部演示工单。

## 闭环流程

- [x] 可以提交新工单。
- [x] 工单列表从 MySQL 读取。
- [x] 工单详情从 MySQL 读取。
- [x] 本地规则分类覆盖账号、权限、系统故障、数据、流程咨询。
- [x] 知识库根据关键词命中。
- [x] 处理建议由模板生成。
- [x] 工单可以人工流转到处理中、已解决、已沉淀。
- [x] 已解决工单可以生成知识草稿。
- [x] 知识草稿必须人工确认后发布。

## 前端

- [x] 第一屏是浅色企业工单辅助处理控制台，不是 landing page。
- [x] 工单队列显示优先级、状态、分类和置信度。
- [x] 当前工单详情显示摘要、上下文、日志、处理记录和时间线。
- [x] 规则/模板建议面板显示分类原因、置信度、知识命中、建议动作和风险提示。
- [x] 知识草稿明确显示“待人工确认”。
- [x] 视觉风格保持浅色、克制、留白、圆角和清晰层级。
- [x] 搜索真实覆盖工单描述、错误日志和知识条目关键词，状态筛选可用。
- [x] 工单创建、状态确认和知识沉淀提供明确成功或错误反馈。
- [x] 表单具备显式标签、键盘焦点态、状态 live region 和跳转主内容链接。
- [x] 截图脚本验证 1366 桌面和 390 移动宽度无横向溢出。

## 安全边界

- [x] 不接真实 LLM。
- [x] 不做自动外部操作。
- [x] 不做自动授权、自动回滚、自动通知。
- [x] 所有规则/模板建议保留“待人工确认”。
