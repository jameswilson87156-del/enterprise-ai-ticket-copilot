# 每次发给 Claude 的审查模板

复制下面整段，把方括号内容替换成刚刚由 Codex 完成的任务。

```text
请在当前项目中先自行审查全项目现状，再进行本轮独立审查。

审查对象：
[粘贴 TODO 名称、相关文件或提交]

审查边界：
- 本轮只审查：[功能或文件]
- 不扩展到：[无关模块]
- 默认只审查，不直接大规模修改代码。

审查步骤：
1. 读取 CLAUDE.md、AGENTS.md、相关 docs、TODO.md、HANDOFF.md、相关源码和测试。
2. 存在 Git 仓库时查看 git status、当前 diff 和相关提交，不只相信 Codex 的总结。
3. 对照 PRD、设计和架构确认实现是否偏离需求。
4. 检查正常流程、加载/空数据/错误/成功状态、回归、安全、隐私和文档一致性。
5. 审核 Codex 声称的测试证据；必要时运行现有测试或构建命令。
6. 没有执行的验证要明确说明，不得编造“已通过”。

请把审查问题追加到 HANDOFF.md。每个问题必须包含：
- 文件和具体位置；
- 严重程度（高/中/低）；
- 证据或复现步骤；
- 实际结果与预期结果；
- 建议修改方式；
- 交给 Codex 的一个小修复任务；
- 可操作的验收标准。

最后请输出：
- 审查过的文件；
- 实际执行的验证和结果；
- 问题数量及严重程度；
- 是否建议验收当前 TODO；
- 应最先交给 Codex 的唯一修复任务。
```
