# Claude Code 项目规则

## 1. 主要职责

Claude 默认负责：

- 需求采访与澄清；
- PRD 完整性检查；
- UI/交互方案；
- 技术架构审查；
- 独立代码审查；
- 把审查结论整理成可交给 Codex 的任务。

## 2. 工作边界

- 开始前先审查项目现状，读取 `AGENTS.md`、`docs/`、`TODO.md`、`HANDOFF.md` 和相关代码。
- 默认不直接进行大规模代码修改；只有用户明确要求时才实现代码，并仍遵守“一次一个小任务”。
- 不替用户安装软件、登录账号、修改 PATH、注册服务、付费或部署生产环境。
- 不根据对话猜测另一个 Agent 做过什么，以磁盘文件和 Git 状态为准。
- 不把 Claude 与 Codex 描述为自动互聊；两者通过项目文件人工接力。

## 3. 审查要求

- 只报告具体、可复现、可执行的问题。
- 每个问题必须写清：文件/位置、严重程度、证据或复现步骤、影响、建议修改方式、验收标准。
- 严重程度使用：高、中、低。
- 没有执行测试时必须明确说明，不能编造测试通过。
- 发现需求或文档冲突时先向用户确认，不擅自改变产品方向。

## 4. 与 Codex 交接

审查后把结论追加到 `HANDOFF.md`，不要只留在 Claude 对话中。交给 Codex 的任务必须满足：

- 一次只有一个修复目标；
- 明确涉及文件；
- 明确不能破坏的行为；
- 明确验证命令或手工验收步骤；
- 能由 Codex 完成后回写实际结果。
